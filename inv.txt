    31 CItem番号
       NOTE: この番号はセーブデータ間で異なり、CItemを新しく導入したときなどに書き換わる。CTalkなどでこの変数を直接用いてはならない
    32 CItemハッシュ
       CItem の ID (文字列) から計算されたハッシュ値
       NOTE: ハッシュ値はその性質上「絶対に」被ることを避けられない (事実上無限とみなせる文字列の ID に対し、このハッシュ値は高々32bitしかないため)。事実、現時点でさえこのハッシュ値が被っているCItemが存在する。inv(31, *) はセーブデータロード時に適切なものに置き換えられるが、その際、CItem の同一性はこのハッシュ値から判定される。この問題は 2020.5.12 現在、omake系列全てにおいて未解決であり、ワークアラウンドとして既存の CItem とハッシュ値が被らないような名前をつけることが推奨される
    33 アイテムに関連付けられたCNPC番号
       通常は、本家の inv で本家のキャラID が代入されていたもの (モンスターボールの中身など) を CNPC にも拡張するために用いられている
    34 アイテムに関連付けられたCNPCハッシュ
       CItem の ID (文字列) から計算されたハッシュ値
       NOTE: inv(32, *) と同じ問題を抱える。詳しくは上述の通り
    39 アイテムの方向指定
       地面に置かれていないアイテムは常に 0。方向指定差し替えについては omake の readme に詳しい
       0: 南 (デフォルト)
       1: 西
       2: 東
